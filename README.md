# web-presentation

基于 reveal.js 的网页演示制作 Skill。用 HTML 网页开发代替传统 PPT，让 AI 做出真正好看的演示文稿。

---

## 为什么做这个

我试过让 AI 帮我做 PPT——用的还是目前效果最好的 Kimi。结果发现：

- **贵**，额度还少，做几份就用完了
- **只能一次性生成**，发了需求就等着，出来什么样就是什么样
- **不能选模板**，风格完全不可控
- **不能逐页改**，想调整第三页？只能重新说一遍需求碰运气

本质原因是：AI 擅长处理文字描述清楚的任务，但对 GUI 界面的操控还很弱。直接让 AI 操作 PowerPoint，它做不到精细控制。

**换一个思路：** AI 的网页开发能力已经非常强了。用网页来模拟 PPT，等于让 AI 在它最擅长的领域做演示。测试之后效果远超预期——动画、布局、视觉效果都远好于直接生成 PPT 文件。

## 我们的优势

| 对比 | 传统 PPT Agent | web-presentation |
|------|---------------|------------------|
| **费用** | 按次付费，额度有限 | **完全免费**，调用你自己的大模型 API |
| **修改方式** | 一次性生成，改了就重来 | **逐页调整**，改第三页不影响其他页 |
| **模板** | 固定几种，不可自定义 | **支持自定义模板**，Agent 引导你创建 |
| **视觉效果** | 受 PowerPoint 能力限制 | **CSS 动画 + 网页渲染**，效果上限高得多 |
| **交互** | 发完需求就等 | **先沟通确认方案**，再动手制作 |

## 核心特点

### 📄 分页架构，按你的想法改

每张幻灯片是独立的 `pages/XX-name.html` 文件。想改第三页？直接改那个文件，不影响其他页面。不用在一个几千行的 HTML 里翻找。

```
pages/
├── 01-title.html      ← 封面
├── 02-background.html ← 背景介绍
├── 03-method.html     ← 方法（想改这页？改这个文件就行）
├── 04-results.html    ← 结果
└── 05-ending.html     ← 结尾
```

**和 Agent 的协作方式：**
- "把第三页的标题改成 xxx" → Agent 直接改 `03-method.html`
- "在方法页后面加一页图表" → Agent 新建页面文件，更新 manifest
- "删掉第五页" → Agent 删除文件，更新 manifest

每页改动彼此独立，不会破坏其他页面。

### 🎨 多模板，可自定义

内置 4 种模板（学术 / 科技 / 项目汇报 / 极简），也支持创建自己的风格。

| 模板 | 适用场景 | 风格 |
|------|----------|------|
| `academic` | 学术报告、论文答辩 | 白底深蓝，严谨 |
| `tech-talk` | 技术分享、演讲 | 深色背景，科技感 |
| `project-review` | 项目汇报、进度展示 | 灰白简洁 |
| `minimal` | 极简风格 | 无装饰 |
| *你的模板* | *任何场景* | *完全自由* |

**想创建自己的模板？** 参见 [templates/CUSTOM-TEMPLATE-GUIDE.md](templates/CUSTOM-TEMPLATE-GUIDE.md)，或直接告诉 Agent 你想要的风格，TA 会引导你完成。

### 📎 从现有 PPT 提取模板

已有做得不错的 PPT？提供 .pptx 文件，Agent 会提取它的设计风格生成模板：

1. 运行提取脚本，解析配色、字体、布局、动画等结构化数据
2. 结合截图做视觉分析（氛围、对齐、留白等）
3. 综合生成 `模板.md`
4. 你确认后，用这个模板生成新演示

```
"参考这个 PPT 的风格帮我做一个新的"
"我喜欢这个模板，帮我提取出来"
```

详细说明见 [templates/EXTRACT-FROM-PPT.md](templates/EXTRACT-FROM-PPT.md)。

### 🤝 制作前先沟通

不会拿到需求就直接开做。Agent 会先确认你的需求（受众、时长、关键内容），匹配模板，展示方案大纲，等你确认后再动手。

### 📄 最低标准，最大自由

系统提示词只保证底线品质（可读性、内容不溢出、视觉一致性），不强制布局、配色、动画的具体选择。你的需求和选择的模板才是最终指南。

设计优先级：`用户需求 > 模板偏好 > 系统默认参考`

## 设计理念

**最低标准，最大自由。**

系统提示词提供最低美学基线，确保每份演示都有基本的专业品质。但不强制具体的布局、配色、动画——这些由**模板**和**用户需求**决定。

## 功能一览

- 📄 分页架构 — 每个页面独立编辑，互不影响
- 🎨 多模板支持 — 学术风 / 科技风 / 项目汇报 / 极简 / 自定义
- 📎 PPT 提取模板 — 从现有 .pptx 提取设计风格，生成可复用模板
- 📐 16:9 宽屏（默认），支持 4:3
- ✅ 内容溢出防护 — 内容不超出画布
- 📄 自动导出 PDF — 高分辨率（2x deviceScaleFactor）
- 🖥️ 本地预览 — 自动启动 dev-server 并打开浏览器
- ✍️ KaTeX 公式 + 代码语法高亮
- 🤝 主动沟通 — 制作前确认方案，不盲目动手
- 🎨 自定义模板 — Agent 引导式帮你创建专属风格
- 🌐 网络素材检索 — 自动从 Unsplash/Pexels 获取专业图片，Lucide Icons 零配置图标

## 快速使用

在对话中直接说：

```
帮我做一个关于"量子计算入门"的PPT
```

Agent 会：
1. 确认需求细节（目标受众、页数、关键内容）
2. 匹配模板，呈现方案供确认
3. 确认后初始化项目，生成 `pages/*.html`
4. 自动启动本地服务器，打开浏览器预览
5. 完成后导出 PDF

**修改演示：**

```
把第三页的标题改一下
在方法页后面加一页图表
换成科技风的模板
```

## 项目结构

```
projects/<project-name>/
├── index.html          ← 开发版（fetch 动态加载 pages）
├── dev-server.py       ← 本地预览服务器
├── manifest.json       ← 项目元数据 + 页面列表
├── project-index.md    ← 项目文档
├── style.css           ← 全局样式
├── assets/             ← 图片、数据等资源
├── pages/              ← 每个幻灯片页面（独立文件）
│   ├── 01-title.html
│   ├── 02-content.html
│   └── ...
└── build/              ← 构建产物（内联单文件，用于分发）
    └── index.html
```

最终交付：`output/<project-name>/` 下的 HTML + PDF。

## 预览

```bash
cd <project-dir>
python3 dev-server.py        # 默认 8000 端口
python3 dev-server.py 8080   # 自定义端口
```

浏览器打开 `http://localhost:8000`。

## 依赖

- Node.js ≥ 16（PDF 导出需要 puppeteer）
- Python 3（本地服务器）
- 现代浏览器

依赖检查：`bash scripts/check_deps.sh`

## 网络素材检索

内置自动化素材获取能力，全程无需人工参与。Agent 可独立从互联网检索并引用专业视觉素材。

**推荐来源：** Unsplash / Pexels（壁纸背景）、Lucide Icons（图标）、Lorem Picsum（占位图）、CSS 渐变 / 内联 SVG（装饰）。

**核心原则：** 专业素材优先替代 emoji 占位，仅使用免费可商用资源，全自动化无需注册登录。

详见 SKILL.md 中的「🌐 网络素材检索」章节。
