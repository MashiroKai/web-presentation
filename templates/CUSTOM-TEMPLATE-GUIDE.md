# 自定义模板指南

> 你的演示，你的风格。这份指南帮助你从零创建专属模板。

---

## 模板是什么

模板是一个 Markdown 文件，定义演示的**视觉风格**——配色、字体、布局偏好、动画风格。它不控制内容，只控制"看起来怎样"。

内置模板（academic、tech-talk、project-review、minimal）是起点，不是终点。你可以创建完全属于自己的风格。

---

## 创建方式

### 方式一：跟 Agent 说

直接告诉 Agent 你想要的风格：

```
我想创建一个自己的模板，风格是...
- 主色调：我喜欢橙色系
- 整体感觉：温暖、有活力，适合面向非技术受众的分享
- 布局：喜欢左对齐，不要太居中
- 动画：稍微活泼一点，可以用 scale 效果
```

Agent 会：
1. 理解你的审美偏好
2. 推荐最接近的内置模板作为基础
3. 生成符合你风格的模板文件
4. 解释每个部分的作用，让你确认或调整

### 方式二：手动创建

1. 阅读本文档的结构说明
2. 复制一个最接近的内置模板（如 `academic.md`）
3. 逐节修改，保存到 `templates/` 文件夹

## 模板结构

每个模板必须包含以下部分。可以增减，但建议保持完整以确保 Agent 能正确理解和应用你的模板。

### 1. 标题与场景

```markdown
# 模板：我的风格

## 场景
适合什么类型的演示？（学术、技术、商业、个人分享……）
```

场景描述帮助 Agent（和你自己）判断什么时候用这个模板。

### 2. 设计语言

```markdown
## 设计语言
用一句话定义这个模板的审美气质。
```

这是模板的灵魂。好的设计语言描述应该让人一眼感受到风格方向。

**示例：**
- "严谨、克制、值得信赖"（academic）
- "暗色、专注、沉浸"（tech-talk）
- "空即是满"（minimal）
- "温暖、友好、有呼吸感"（假设的暖色模板）
- "赛博朋克，霓虹灯下的数据"（假设的科幻模板）

### 3. 配色

```markdown
## 配色

| 语义 | 色值 | 用途 |
|------|------|------|
| 主色 | `#FF6B35` | 标题、强调 |
| 辅助色 | `#FFA07A` | 注释、页码 |
| 强调色 | `#2E4057` | 关键数据 |
| 正文色 | `#333333` | 正文文字 |
| 背景 | `#FFF8F0` | 页面底色 |
| 深色背景 | `#2E4057` | 封面、结束页 |
```

**配色建议：**
- 选 2-3 个主色即可，不需要太多
- 深色背景上的文字用浅色（不要纯白 `#FFFFFF`，用柔和的浅色）
- 文字色不要用纯黑（`#000000`），用深灰
- 强调色要和主色有足够对比度
- 不确定就先参考内置模板，微调色相

**配色灵感来源：**
- 你的品牌色 / Logo 颜色
- 你喜欢的网站或 App 的配色
- [Coolors.co](https://coolors.co) — 随机配色生成器
- [Realtime Colors](https://www.realtimecolors.com/) — 实时预览配色效果

### 4. 字体

```markdown
## 字体

| 元素 | 字体 | 字重 |
|------|------|------|
| 标题 | Helvetica Neue Bold, Arial Bold | 700 |
| 正文 | Helvetica Neue, Arial | 400 |
| 代码 | SF Mono, Menlo, Consolas | 400 |
```

**字体建议：**
- 标题和正文用同一个字体族的不同字重，最安全
- 代码必须用等宽字体
- 中文优先选苹方（macOS）或微软雅黑（Windows）
- 不确定就用系统默认字体栈：`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`

### 5. 布局偏好

```markdown
## 布局偏好
- **封面** — 怎样设计封面？
- **内容页** — 居中？左对齐？分栏？
- **过渡页** — 章节页怎么处理？
- **结束页** — 怎样收尾？
```

**布局选项：**
- 居中（最安全、通用）
- 左对齐（适合密集内容、阅读型）
- 分栏 / Flex（适合图文并排）
- 全幅背景图（适合视觉冲击）
- 没有标准答案，选你觉得舒服的

### 6. 动画风格

```markdown
## 动画风格
- **进入** — fade-in / 无动画 / 其他
- **强调** — highlight / scale / 其他
- **页间过渡** — fade / slide / zoom / 其他
- **限制** — 每页最多几个 fragment？
```

**动画选项：**
- `fade-in` — 最安全，几乎不会出错
- `highlight-red` / `highlight-green` — 强调关键信息
- `grow` / `shrink` — 放大/缩小（节制使用）
- 页间过渡：`fade`（推荐）、`slide`（左右滑动）、`zoom`（缩放）、`convex`（3D 翻转）

### 7. 特殊要求（可选）

```markdown
## 特殊要求
- 支持 LaTeX 公式
- 支持代码高亮（monokai 主题）
- 页码样式
- Logo 位置
- 其他自定义需求
```

## 完整示例

以下是一个假设的"温暖商务风"模板，展示完整的结构：

```markdown
# 模板：温暖商务

## 场景
面向非技术受众的商业演示、产品介绍、团队分享

## 设计语言
温暖、亲和、有活力。用暖色调传递友好感，但不失专业。适合需要拉近距离、建立信任的场景。

## 配色

| 语义 | 色值 | 用途 |
|------|------|------|
| 主色 | `#FF6B35` | 标题、按钮 |
| 辅助色 | `#FFA07A` | 副标题、装饰 |
| 强调色 | `#2E4057` | 关键数据 |
| 正文色 | `#333333` | 正文 |
| 背景 | `#FFF8F0` | 页面底色 |
| 深色背景 | `#2E4057` | 封面、结束页 |

## 字体

| 元素 | 字体 | 字重 |
|------|------|------|
| 标题 | Nunito Bold, -apple-system | 700 |
| 正文 | Nunito, -apple-system | 400 |
| 代码 | Fira Code, Menlo | 400 |

## 布局偏好
- **封面** — 居中，深色背景，大标题 + 一句话副标题
- **内容页** — 左对齐标题，内容区域居中，要点之间留白充足
- **数据页** — 图表居中，结论在标题
- **过渡页** — 居中大字，暖色背景
- **结束页** — 深色背景，"Thank You" + 联系方式

## 动画风格
- **进入** — fade-in
- **强调** — highlight-red 用于关键数字
- **页间过渡** — fade
- **限制** — 每页 ≤ 3 个 fragment

## 特殊要求
- 支持图表（Chart.js）
- 页码右下角，辅助色
```

## 常见问题

### Q: 我不知道选什么配色怎么办？

告诉 Agent 你的需求：
- "我的公司主色是蓝色"
- "我想要暗色系，类似 GitHub 的风格"
- "我想用莫兰迪色系"

Agent 会帮你推荐。

### Q: 我可以在已有模板基础上改吗？

当然。复制一个最接近的内置模板，修改你想要的部分就行。没改的部分保持原样。

### Q: 我创建的模板能复用吗？

能。模板文件保存在 `templates/` 文件夹，创建新演示时可以引用已有的自定义模板。

### Q: 需要写 CSS 吗？

不需要。模板是 Markdown 格式，只描述风格意图。Agent 会根据模板描述生成实际的 CSS。但如果你懂 CSS，也可以在模板中直接写 CSS 变量。

### Q: 模板格式有严格要求吗？

没有。上述结构是推荐格式，但你可以增减。核心是让 Agent（或未来的你）能理解这个模板想要的风格。
