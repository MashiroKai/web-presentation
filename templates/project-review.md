# 模板：项目汇报

## 场景
项目进度汇报、团队评审、阶段性总结、里程碑回顾

## 设计语言
专业、清晰、数据驱动。用绿色表达正向进展，红色标注风险。整体风格稳重，强调信息的可扫描性——评审者应该在 3 秒内抓住每页重点。

## 配色

| 语义 | 色值 | 用途 |
|------|------|------|
| 主色 | `#1D1D1F` | 标题、正文 |
| 辅助色 | `#86868B` | 注释、页码 |
| 正向色 | `#34C759` | 完成项、达标数据 |
| 风险色 | `#FF3B30` | 延期、风险项 |
| 警告色 | `#FF9500` | 需关注项 |
| 背景 | `#FFFFFF` | 页面底色 |

## 字体

| 元素 | 字体 | 字重 |
|------|------|------|
| 标题 | SF Pro Display Bold, -apple-system | 700 |
| 正文 | SF Pro Text, -apple-system | 400 |
| 数据 | SF Pro Text, -apple-system | 600 |
| 代码 | SF Mono, Menlo | 400 |

## 布局偏好
- **封面** — 项目名 + 汇报人 + 日期，居中，深色背景
- **数据页** — 图表占 50%+ 面积，结论在标题中体现
- **进度页** — 时间线或甘特图风格，绿/红标注状态
- **对比页** — 左右分栏：计划 vs 实际
- **总结页** — 3–5 条核心结论，大字展示

## 动画风格
- **进入** — 淡入 (fade-in)
- **强调** — `fragment highlight-green` / `fragment highlight-red` 标注状态
- **页间过渡** — fade
- **限制** — 稳重克制，每页 ≤ 3 个 fragment

## 特殊要求
- 数据图表优先（Chart.js / 表格）
- 支持进度条、状态标签等 UI 组件
- 正向数据用绿色，风险用红色，语义固定
- 页码右下角
