# Templates Guide — 模板系统说明

## 模板是什么

模板是一个 Markdown 文件，定义演示的**视觉风格**。不强制统一，按场景按需加载。模板控制"看起来怎样"，不控制"讲什么"。

---

## 模板结构

```markdown
# 模板名称

## 场景
适用场景描述

## 设计语言
一句话定义这个模板的审美气质

## 配色
| 语义 | 色值 | 用途 |

## 字体
| 元素 | 字体 | 字重 |

## 布局偏好
各页面类型的默认布局和约束

## 动画风格
- 进入: 效果
- 过渡: 效果
- 限制: 规则

## 特殊要求
页码、Logo、公式支持等
```

---

## 内置模板

| 模板 | 文件 | 场景 | 气质 |
|------|------|------|------|
| **academic** | `templates/academic.md` | 学术汇报、论文答辩 | 严谨、克制、值得信赖 |
| **tech-talk** | `templates/tech-talk.md` | 技术分享、会议演讲 | 暗色、专注、沉浸 |
| **project-review** | `templates/project-review.md` | 项目汇报、团队评审 | 专业、清晰、数据驱动 |
| **minimal** | `templates/minimal.md` | 极简风格、通用 | 空即是满 |

### 选择指南

```
用户需求
  ├─ 学术/论文/答辩 → academic
  ├─ 技术/代码/演讲 → tech-talk
  ├─ 项目/进度/评审 → project-review
  └─ 通用/极简/概念 → minimal
```

---

## 自定义模板

1. 复制一个最接近的内置模板作为起点
2. 修改配色、字体、布局偏好
3. 添加「设计语言」一句话描述审美气质
4. 保存为 `custom.md` 或任意名称
5. 在 manifest.json 中引用

---

## 模板应用流程

```
选择模板 → 读取模板 MD
    ↓
提取配色 → 生成 style.css CSS 变量
    ↓
提取动画偏好 → 配置 Reveal.js 参数
    ↓
提取布局偏好 → 指导页面结构生成
```

模板不控制内容，只控制视觉呈现。
